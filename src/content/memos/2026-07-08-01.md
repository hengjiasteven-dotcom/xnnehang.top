在运行 `yutto` 进行扫码登录时，若遇到连接重置等代理或网络环境问题，需要在登录及其中间交互阶段也同样配置不使用代理：

```shell
D:\lab\yuttos\uiya-yutto>uv run yutto auth login --proxy no
 INFO  请使用哔哩哔哩 App 扫码并确认登录
 INFO  二维码待扫描
 WARN  请求登录确认 URL 失败，将尝试从返回 URL 提取 cookies：[WinError 10054] 远程主机强迫关闭了一个现有的连接。
 INFO  登录成功，已写入认证文件：C:\Users\Administrator\AppData\Roaming\yutto\auth.toml（profile: default，url: https://passport.biligame.com/x/passport-login/web/crossDomain）
```

`--proxy no` 不仅仅在解析下载的时候需要加上，`auth login` 以及 login 途中涉及的操作也许都要加上。
